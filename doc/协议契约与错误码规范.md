# AhaAgent 协议契约与错误码规范（V1）

## 1. 目的与适用范围

本文档定义 AhaAgent V1 的前后端通信协议、工具调用契约、幂等语义与错误码规范。

适用对象：

- Web Client
- Daemon Core
- AI 开发 Agent（代码生成与测试）

## 2. 协议总则

1.  所有 WS 消息必须使用统一信封（Envelope）。
2.  所有客户端请求必须包含 `requestId` 与 `idempotencyKey`。
3.  服务端对重复 `idempotencyKey` 必须返回同一语义结果，不得重复执行破坏性动作。
4.  事件与字段必须向后兼容，破坏性改动需要升级 `protocolVersion`。

## 3. WebSocket 消息信封

### 3.1 TypeScript 契约

```ts
interface WsEnvelope<T> {
  protocolVersion: '1.0';
  sessionId: string;
  requestId: string; // UUIDv7 推荐
  idempotencyKey: string; // 相同语义请求必须复用同一 key
  timestamp: string; // ISO8601
  type: string; // 事件名
  payload: T;
}
```

### 3.2 JSON Schema（可直接给 AI 生成校验器）

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "WsEnvelope",
```

## 5. 工具调用契约（Tool Contract）

### 5.1 通用结构

```ts
interface ToolCall<TInput> {
  toolName: string;
  input: TInput;
}

interface ToolResult<TOutput> {
  ok: boolean;
  output?: TOutput;
  errorCode?: string;
  errorMessage?: string;
}
```

### 5.2 核心工具输入输出

#### `read_file`

- 输入：`{ path: string }`
- 输出：`{ path: string; content: string; version: string; sensitivity: 'public' | 'restricted' | 'secret' }`

#### `list_dir`

- 输入：`{ path: string; depth?: number }`
- 输出：`{ entries: Array<{ name: string; path: string; type: 'file' | 'dir' }> }`

#### `grep`

- 输入：`{ pattern: string; path: string; glob?: string }`
- 输出：`{ matches: Array<{ file: string; line: number; text: string }> }`

#### `diff_edit`

- 输入：`{ path: string; expectedVersion: string; hunks: Array<{ oldText: string; newText: string }> }`
- 输出：`{ path: string; version: string; appliedHunks: number }`

#### `write_file`

- 输入：`{ path: string; expectedVersion?: string; content: string }`
- 输出：`{ path: string; version: string }`

#### `run_command`

- 输入：`{ command: string; args: string[]; cwd?: string; timeoutSec?: number }`
- 输出：`{ exitCode: number; stdout: string; stderr: string }`

#### `browser_search`

- 输入：`{ query: string; engine?: 'duckduckgo'; maxResults?: number }`
- 输出：`{ results: Array<{ id: string; title: string; url: string; snippet: string }> }`

#### `fetch_url` / `browser_open`

- 输入：`{ url: string; timeoutSec?: number; maxBytes?: number }`
- 输出：`{ finalUrl: string; status: number; title: string; text: string; html?: string }`

#### `extract_main_content`

- 输入：`{ html: string }`
- 输出：`{ title: string; markdown: string }`

#### `browser_tool`（浏览器自动化统一入口）

- 输入：
  - `action: 'status' | 'start' | 'stop' | 'search' | 'open' | 'click_result' | 'click' | 'type' | 'snapshot'`
  - 其它字段按 action 选择：`query/engine/maxResults/url/index/selector/text/submit/timeoutSec/maxChars`
- 输出：根据 action 返回对应结构化结果（如当前 URL、页面标题、搜索结果、页面快照等）。
- 说明：
  - `browser_tool` 是编排层工具名，在策略层映射为只读动作 `browser_open`。
  - 浏览器导航必须通过 URL 安全校验（协议、localhost/内网拦截）。

#### `memory_search`（记忆搜索）

- 输入：`{ query: string; topK?: number; category?: 'preference' | 'fact' | 'skill' | 'context' }`
- 输出：`{ ok: true; output: string }` — output 为 JSON 数组 `[{ id, content, category }]`
- 说明：
  - Agent 自主调用，用于在回答问题前检索历史记忆。
  - CJK 查询使用 jieba 分词 + LIKE 匹配；非 CJK 使用 FTS5 BM25 排序。
  - 自动排除 `secret` 敏感记忆；`restricted` 需显式开启。
  - 命中记忆的 `access_count` 和 `last_accessed_at` 自动更新。
  - 策略层视为只读操作，无需审批。

#### `memory_store`（记忆存储）

- 输入：`{ content: string; category: 'preference' | 'fact' | 'skill' | 'context'; sensitivity?: 'public' | 'restricted' | 'secret' }`
- 输出：`{ ok: true; output: string }` — output 为 JSON `{ id: "..." }`
- 说明：
  - Agent 自主调用，用于存储值得长期保留的信息。
  - 写入后自动同步 FTS5 索引。
  - 写入后检查总条目数，超过 `AHA_MEMORY_MAX_ENTRIES` 时自动淘汰低价值记忆。
  - 策略层视为内部操作，无需审批。

## 6. 幂等规则

1.  `send_message`：同 `idempotencyKey` 且同 `conversationId`，服务端仅保留一次任务创建。
2.  `approve_action`：`approvalId + approvalNonce` 只能成功一次。
3.  `cancel_task`：重复取消返回同一终态，不重复触发清理逻辑。
4.  `write_file/diff_edit/delete_file/run_command/install_extension`：必须使用幂等键防重。

## 7. 错误码规范

### 7.1 命名格式

`AHA-<DOMAIN>-<3_DIGIT>`，例如 `AHA-AUTH-001`。

### 7.2 领域划分

- `AUTH`：认证/鉴权
- `POLICY`：策略拒绝/审批要求
- `SANDBOX`：路径与权限沙箱
- `TOOL`：工具执行错误
- `TASK`：任务状态与并发错误
- `EXT`：扩展安装/运行错误
- `LLM`：模型调用错误
- `SYS`：系统级未知错误

### 7.3 V1 必选错误码

| 错误码            | 含义                    | retryable | 处理建议            |
| ----------------- | ----------------------- | --------- | ------------------- |
| `AHA-AUTH-001`    | sessionToken 缺失或无效 | false     | 重新建立会话        |
| `AHA-AUTH-002`    | Origin/Host 校验失败    | false     | 拒绝请求并告警      |
| `AHA-POLICY-001`  | 动作被策略拒绝          | false     | 调整策略或改方案    |
| `AHA-POLICY-002`  | 动作需要审批            | false     | 等待用户审批        |
| `AHA-SANDBOX-001` | 路径越权/逃逸           | false     | 修改目标路径        |
| `AHA-SANDBOX-002` | 访问敏感文件被拒绝      | false     | 改为脱敏路径        |
| `AHA-TOOL-001`    | 工具参数非法            | false     | 修正参数            |
| `AHA-TOOL-002`    | 文件版本冲突            | true      | 重新读取后重试      |
| `AHA-TASK-001`    | 任务不存在或已结束      | false     | 刷新任务状态        |
| `AHA-TASK-002`    | 锁冲突                  | true      | 退避重试            |
| `AHA-EXT-001`     | 扩展签名/校验失败       | false     | 禁止安装            |
| `AHA-EXT-002`     | 扩展运行时崩溃          | true      | 重启隔离 Runner     |
| `AHA-LLM-001`     | 上游模型请求失败        | true      | 指数退避重试        |
| `AHA-SYS-001`     | 未知系统错误            | true      | 记录 traceId 后重试 |

## 8. 兼容性与版本策略

- `protocolVersion=1.0` 为 V1 固定版本。
- 新增字段只能“可选新增”，不得删除/重命名现有必填字段。
- 破坏性升级必须进入 `2.0`，并提供迁移说明。

## 9. AI 开发执行要求

1.  先生成 Envelope 与事件 schema 校验中间件，再写业务代码。
2.  所有 handler 必须先做 `requestId/idempotencyKey` 校验。
3.  错误处理统一映射到本文档错误码，禁止自造错误格式。

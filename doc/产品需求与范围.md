# AhaAgent 产品需求与范围（V1）

## 1. 产品目标

AhaAgent 是本地运行的 AI Coding Agent，用户通过 Web 页面下达自然语言命令，系统自动完成：

- 代码阅读与修改
- 命令执行与验证
- 审批拦截与恢复执行
- 网页检索与浏览器自动化辅助

目标是实现“可用、可控、可审计、可恢复”的本地开发助手。

## 2. V1 范围

### 2.1 必须实现

- 本地聊天 UI（任务状态、审批弹窗、开发者控制台）
- 工具链：`read_file`、`list_dir`、`diff_edit`、`write_file`、`delete_file`、`run_command`
- Web 能力：`browser_search`、`fetch_url`、`browser_open`、`extract_main_content`
- 浏览器自动化：`browser_tool`（`start/status/search/open/click_result/click/type/snapshot/stop`）
- 安全基线：工作区沙箱、敏感数据防泄漏、高危动作审批
- 编排闭环：工具调用 -> 审批 -> 恢复继续 -> 终态回传

### 2.2 非目标

- 模型自主安装第三方扩展并直接执行
- 无约束自动外发本地敏感内容到外部模型
- 把不可信扩展直接加载到主进程作为安全边界

## 3. 关键用户体验

1. 用户只需输入自然语言，不需要手工勾选工具。
2. 高危动作自动触发审批，审批后任务自动恢复执行。
3. 后端终端可看到任务步骤、工具调用与结果，避免“卡住不可观测”。
4. 浏览器自动化默认可见窗口，便于观察执行过程。

## 4. 版本能力说明（截至 2026-03-06）

### 4.1 已交付

- Agent 自动循环与审批恢复闭环
- 基础文件/命令工具链
- Web 搜索与 URL 抓取
- `browser_tool` 浏览器自动化（Playwright）
- 浏览器会话模式：
  - 默认启动可见浏览器（`AHA_BROWSER_HEADLESS=0`）
  - 可通过 `AHA_BROWSER_CDP_URL` 连接外部浏览器

### 4.2 已知限制

- Daemon 重启后的自动 checkpoint 恢复尚未完全闭环
- MCP 扩展协议通信层仍在完善
- 复杂登录/反爬站点的浏览器自动化成功率受站点策略影响

## 5. 安全与治理原则

1. 默认拒绝：权限、扩展、外发均默认拒绝。
2. 显式授权：`write_file/delete_file/run_command/install_extension` 必须审批。
3. 最小权限：任务、工具、扩展仅获最小必要权限。
4. 可审计：关键动作写入审计日志，可按 `traceId/requestId/taskId` 追踪。

## 6. 里程碑建议

1. 完善 checkpoint 重启恢复与回放能力。
2. 完成 MCP 协议通信闭环（安装、授权、调用、审计）。
3. 增强浏览器自动化稳定性（重试、等待策略、结果校验）。
